Open data cloud

This is the data platform which integrates open data from the government or related institutes for internal employee use. The main purpose is to address the inconvenience that employees have to visit individual websites or webpages to manually download the latest data. Additionally, for most routine jobs, employees need to download different data frequently, which can be quite burdensome for them.

Therefore, this platform is hosted on AWS ECS and downloads data regularly with Jenkins, depending on the update frequency. It utilizes AWS Lambda, AWS SQS, AWS EventBridge, AWS SES, AWS CloudWatch, AWS Step Functions, and AWS DynamoDB to perform a series of actions including ETL, processing, and organization. Finally, the organized data is stored in AWS S3.

In this project, I was the main developer and project owner. I developed over 30 web crawlers using Python and maintained various AWS services. Additionally, I designed and implemented anomaly detection and operational alerts within AWS CloudWatch to ensure system reliability. My responsibilities also included optimizing crawler performance, managing data pipelines, and integrating new features based on user feedback. I collaborated closely with cross-functional teams to enhance the overall functionality and scalability of the project.
